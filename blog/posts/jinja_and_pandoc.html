Why Jinja2 and Pandoc

I also had no idea about these two tools until I started working on my little own personal blog project. All I wanted was to automate the process to create static web pages based on a given template, and host them as GitHub pages. Normally, you can easily create templates and render them on the server using backend frameworks in response to user requests. But this is already too complex for a simple personal blog—let alone the fact that it requires a dedicated server, backend management, and paid hosting and domain services.

If only there was a light-weight tool that can do the server site rendering without having to run an actual server and most of its services which are totally uncessary for this task. Fortunatelly, such a tool already exists and it is Jinja2. It simply allows you to use placeholders in a file and fill them in later based on your logic. You can implement this logic in Python and pass the resulting values to the placeholders using the Jinja2 package. Below is a simple example to get you started.

Jinja2 Template

  • Uses {% ... %} for logic (if, for, etc.)
  • Uses {{ ... }} for variables

Example

from jinja2 import Template

# Define the template
template = Template("""
{% if title %}
  <h1>{{ title }}</h1>
{% endif %}
<ul>
{% for post in posts %}
  <li>{{ post.title }}</li>
{% endfor %}
</ul>
""")

# Variables to inject into the template
vars = {
    "title": "Jinja2 and Pandoc",
    "posts": [
        {"title": "Understanding Jinja2"},
        {"title": "Markdown to HTML with Pandoc"},
        {"title": "Template Engines Overview"}
    ]
}

# Render the template
rendered = template.render(vars)

You might be wondering, “Why do I need Pandoc then?” While Jinja2 is great for filling in templates, it doesn’t handle converting Markdown to other formats like HTML or PDF. That’s where Pandoc comes in. Pandoc is a powerful document converter that can transform files between many formats, including Markdown (.md), HTML (.html), PDF (.pdf), and LaTeX (.tex).

Even after using Jinja2 to fill in your placeholders, you would still need to write additional code to convert your Markdown files to HTML or other formats. This process becomes even more complex if your content includes LaTeX syntax, since you’d need to integrate a LaTeX compiler to render those parts correctly. Pandoc simplifies all of this by handling the conversion for you. It also supports its own templating system (though less flexible than Jinja2), allowing you to customize the output format with CSS and JavaScript.

Pandoc uses a slightly different syntex for templating as illustrated below.

Pandoc Template

  • Variables use $...$
  • Conditionals use $if(...)$ ... $endif$
  • Loops use $for(...)$ ... $endfor$

Example

$if(title)$
  <h1>$title$</h1>
$endif$
$for(posts)$
  <li>$posts.title$</li>
$endfor$
